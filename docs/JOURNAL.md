# JOURNAL — 决策日志

> 只记**为什么**，不记"做了什么"（"做了什么"看 git log）。
> 一条 1-3 行，密度像 commit message。三个月后光看 git log 想不起来的事，才写到这里。

## 写入时机
- 完成一个 playbook step
- 做了一个非显然的技术/产品决定
- 推翻了之前的决定（一定要写，标注"推翻 YYYY-MM-DD 的决定"）

## 不写
- git log 能查到的代码改动
- STATUS / TASKS / LEARNING / ERRORS 已经管的内容
- 日记式碎碎念
